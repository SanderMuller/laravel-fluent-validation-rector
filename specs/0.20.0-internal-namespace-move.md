# 0.20.0 — `Internal\` namespace for `@internal` top-level classes

## Overview

Twice-deferred refactor (originally 0.15.0 Phase 3, then 0.18.0,
then 0.19) lining up the package's `@internal` PHPDoc tags with
the namespace hierarchy. Moves the two top-level `@internal`
classes (`Diagnostics`, `RunSummary`) out of the public namespace
root and into a dedicated `SanderMuller\FluentValidationRector\Internal\`
sub-namespace.

The PHPDoc `@internal` tag is documentation; the namespace move is
structure. Both should agree. Today `Diagnostics::class === 'SanderMuller\FluentValidationRector\Diagnostics'`
sits at the same depth as `FluentValidationSetList::class` (which
*is* public API), with only the PHPDoc tag distinguishing them. A
downstream consumer scanning the namespace tree without reading
docblocks could reasonably mistake `Diagnostics` for public surface.

Ship plan: 0.20.0 (this spec) lands the move + class_alias
deprecation shims at the old locations. 1.0 RC drops the shims.
One minor cycle of warning, two cycles of explicit `@deprecated`
PHPDoc.

## 1. Scope

### In scope

- Move `src/Diagnostics.php` → `src/Internal/Diagnostics.php`
  (new namespace `SanderMuller\FluentValidationRector\Internal`).
- Move `src/RunSummary.php` → `src/Internal/RunSummary.php`
  (same new namespace).
- Update internal consumers: `LogsSkipReasons` trait, rector
  `__construct()` calls, `Tests\` test cases.
- Class-alias deprecation shims at the old locations:
  ```php
  // src/Diagnostics.php (post-move)
  <?php declare(strict_types=1);

  namespace SanderMuller\FluentValidationRector;

  /**
   * @deprecated since 0.20.0 — moved to
   * `SanderMuller\FluentValidationRector\Internal\Diagnostics`.
   * This shim will be removed in 1.0. The class was always
   * `@internal`; the namespace move aligns structure with the
   * PHPDoc tag.
   *
   * @internal
   */
  class_alias(\SanderMuller\FluentValidationRector\Internal\Diagnostics::class, self::class);
  ```
  Equivalent shim for `RunSummary`.
- Update `InternalAuditTest` to recognize `Internal\` namespace as
  `@internal`-by-namespace (no PHPDoc tag required when in
  `Internal\`); allow the two shim files to bypass class scanning
  (they don't define a real class — just trigger the alias).

### Out of scope

- Concerns traits under `Rector\Concerns\` — already namespace-
  scoped to a non-public location. Moving them to `Internal\Rector\Concerns\`
  would be churn for no semantic gain.
- Other internal helpers (`Set\`, `Config\Shared\`) — `Set\` is
  documented public API for `FluentValidationSetList`;
  `Config\Shared\` types like `AllowlistedFactories` are public
  API consumed via DTOs.
- Hard removal of the old namespace. Defer to 1.0 RC.

### Open question — none

OQ #1 (namespace-move shape — `Internal\Diagnostics` vs
`Internal\Diagnostics\Diagnostics` etc.) — resolved in favor of
flat `Internal\Diagnostics` (matches the existing one-class-per-
file convention; nesting deeper for two classes would be
overkill).
